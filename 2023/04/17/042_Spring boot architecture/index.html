Spring boot architecture

Author: Neil Tu Date: April 17, 2023  22:16:00 Category: SpringBoot

Java 專案通常採用三層式架構

包括 Controller、Service 和 Repository，並使用依賴注入（Dependency Injection，DI）技術，以降低它們之間的耦合性。

Controller（控制層）

Controller 負責實踐業務邏輯，包括資料運算、整理等，同時也可以呼叫 Repository 的方法從資料庫中獲取資料。在開發中，Controller 層的資料夾中通常會建立一個 interface 以定義業務功能，然後將其交由一個 class 實作該介面。此外，
在此層也經常會建立多個 DTO（Data Transfer Object）以包裝 PO（Persistent Object）並傳遞資料。

Service（服務層）

Service 層也實踐業務邏輯，包括資料運算、整理等，
同時也可以呼叫 Repository 的方法從資料庫中獲取資料。

Repository（資料訪問層）

Repository 負責與資料庫進行溝通，實現 CRUD（Create、Read、Update 和 Delete）操作。
具體的實作方式取決於使用的 ORM（Object-Relational Mapping）框架。
在此層會建立 DAO（Data Access Object）以及 PO（也被稱為 Entity 或 Bean）。

建立 class 的順序

  1. Entity - 用來表示資料庫中的實體物件，例如一個 User 或是 Product。
  2. DTO (Data Transfer Object) - 用來表示從後端到前端的資料傳輸物件，例如一個 UserDto 或是 ProductDto。
  3. Repository - 用來處理與資料庫的互動，例如一個 UserRepository 或是 ProductRepository。
  4. Interface - 用來定義服務或是資料存取的契約，例如一個 UserService 或是 ProductService 的介面。
  5. Service - 用來實作業務邏輯，例如一個 UserService 或是 ProductService。
  6. Controller - 用來接收 HTTP 請求並回應結果，例如一個 UserController 或是 ProductController。
